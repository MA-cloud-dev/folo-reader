# Folo 极简阅读器 - 需求文档

## 产品定位

一个面向个人用户的 **AI 优先**阅读器，核心理念：

> **先看 AI 提炼，按需深入原文** —— 节省时间

---

## 核心阅读流程

```
订阅源列表  →  AI 摘要卡片流  →  点击展开原文
     │                  │                  │
     │            ————————————           │
     │           |【文章标题】       |          │
     │           | AI: 这篇文章... |          │
     │           | [展开原文] [收藏]|          │
     │            ————————————           │
     └─────────────⚡ 默认视图 ⚡────────────┘
```

**默认视图**：显示 AI 摘要卡片流，而非原文列表

---

## 存储策略

| 内容类型 | 存储策略 | 说明 |
|----------|----------|------|
| 订阅源 | ✅ 永久存储 | URL、名称、分类 |
| 文章元数据 | ✅ 永久存储 | 标题、链接、时间、阅读状态 |
| AI 摘要 | ✅ 永久存储 | 避免重复调用 API |
| 文章原文 | ❌ 不存储 | 按需实时获取 |
| 收藏文章 | ✅ 存储原文 | 用户主动收藏的才保存 |

**设计理由**：
- 不存原文：节省存储空间，避免本地数据膨胀
- 存 AI 摘要：避免重复 API 调用，加快加载速度
- 收藏存原文：重要内容完整保留

---

## 功能优先级

### P0 - 核心功能
- 添加/删除订阅源
- 获取 RSS 并自动生成 AI 摘要
- 摘要卡片流视图
- 点击展开原文

### P1 - 增强功能
- 分类管理
- 收藏功能
- 已读/未读状态

### P2 - 扩展功能
- AI 翻译
- 关键词提取
